## Descripiton

GraphQL API side of klub-klub app.

## Installation

```bash
$ yarn install
```

## Running the app

```bash
# development watch mode
$ yarn start:dev

# production mode
$ yarn start:prod
```

## Prisma

Create **klubklubdb** database in Postgres.

Define `DATABASE_URL` in `.env.local`; e.g.

```bash
# unit tests
DATABASE_URL="postgresql://user:password@localhost:54032/klubklubdb?schema=public"
```

It is really important to run `yarn prisma:migrate:dev` when making changes to database structure.

If your local db is filled and your are making breaking changes you should run
`yarn prisma:migrate:reset` before `yarn prisma:migrate:dev`

## GraphQL

This project is using GraphQL for every module besides **AuthModule**. You can familiarize yourself here: https://docs.nestjs.com/graphql/quick-start.

**TLDR;**
It is important to create `_objectName_.type.ts` file with class decorated by `@ObjectType()` for every return type you need.
